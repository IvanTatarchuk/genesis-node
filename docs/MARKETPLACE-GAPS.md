# Що є на великих маркетплейсах, а в нас поки немає

Порівняння з GPT Store, Hugging Face, Replicate, Zapier/Make app galleries та подібними платформами.

---

## У нас уже є

- **Категорії** — Research, Code, Content, Data, Marketing тощо.
- **Пошук** — за назвою, описом, тегами.
- **Сортування** — популярність, нові, за рейтингом, за ціною.
- **Рейтинг і відгуки** — ★ + кількість, коментарі на сторінці агента.
- **Featured / Boosted** — виділені агенти, буст за кредити.
- **Verified-бейдж** — для агентів (uptime).
- **Профіль креатора** — `/dev/[userId]`, список агентів.
- **Схожі агенти** — на сторінці агента в тій самій категорії.
- **Обкладинка агента** — `cover_image_url` на картці.
- **Теги** — на картці та в фільтрах (`?tag=...`).
- **Підрахунок і пагінація** — скільки агентів знайдено, сторінки.

## Чого немає (ідеї з великих маркетплейсів)

### Discovery і утримання

| Фіча | Що дає | Приклад |
|------|--------|--------|
| **Збережені / Favorites** | Юзер зберігає агентів «на потім»; збільшує повернення. | GPT Store, App Store — «Add to list». |
| **Колекції (підбірки)** | «Staff picks», «For developers», «Trending» — куровані списки. | Hugging Face Collections, Zapier «Popular». |
| **Trending / Rising** | Сортування за останні 7/30 днів, а не тільки all-time popular. | «Trending this week», «Rising stars». |
| **Рекомендації «Для вас»** | На основі історії тасків або обраних категорій. | «Because you used X» / «Similar to what you run». |
| **Сповіщення про нове** | «Нові агенти в Research», «Знижка на агента X». | Email / in-app: нові агенти в обраних категоріях. |

### Довіра і якість

| Фіча | Що дає | Приклад |
|------|--------|--------|
| **Report / Flag** | «Позначити як невідповідний» — модерація, безпека. | У всіх великих маркетплейсах. |
| **Верифікація креатора** | Бейдж «Verified developer» (email/domain/KYC). | App Store, Stripe Verified. |
| **Умови використання агента** | Чітко: дозволене використання, заборона скрапінгу тощо. | Replicate — license per model. |
| **Trust-сигнали на картці** | «Used by X companies», логотипи (якщо є). | Zapier «Used by 500k+». |

### Контент і медіа

| Фіча | Що дає | Приклад |
|------|--------|--------|
| **Скріншоти / демо на картці** | 2–4 зображення або GIF результату — вище CTR. | App Store, Replicate — gallery. |
| **Відгуки з медіа** | Додати скріншот до відгуку. | App Store, Google Play. |
| **Q&A під агентом** | Питання й відповіді від креатора/користувачів. | Amazon, деякі маркетплейси. |
| **Changelog / версії** | «v1.2 — виправлено X, додано Y» — прозорість оновлень. | GitHub Releases, Replicate versions. |

### Монетизація і продукти

| Фіча | Що дає | Приклад |
|------|--------|--------|
| **Підписка на агента** | Місячна «unlimited» замість тільки pay-per-task. | Zapier — тариф за інтеграцію. |
| **Бандли / паки** | «Content pack» — 3 агенти разом зі знижкою. | Steam bundles, app bundles. |
| **Фільтр «Є API»** | Показувати агентів з програмним доступом. | Для B2B та інтеграторів. |

### Платформа і адмін

| Фіча | Що дає | Приклад |
|------|--------|--------|
| **Модерація контенту** | Черга на перевірку нових/скарг, approve/reject. | Усі маркетплейси. |
| **Аналітика для креатора** | Перегляди, кліки, конверсія в таск — вже частково є в dashboard. | Розширити: «views», «clicks», funnel. |

## Що робити першим (пріоритизація)

1. **Report/Flag** — мінімум для безпеки й довіри.
2. **Favorites / збережені агенти** — проста таблиця `saved_agents(user_id, agent_id)` + кнопка на картці/сторінці агента.
3. **Trending за період** — сортування за `tasks_last_7d` або окремий індекс; блок «Trending this week» на маркетплейсі.
4. **Скріншоти на картці** — 1–4 зображення в картці агента (поле `gallery_urls[]` або окрема таблиця).
5. **Колекції** — спочатку статичні: «Staff picks», «For developers» (фільтр по одній колекції з адмінки).

Далі — verified developer, changelog, Q&A за потребою.
